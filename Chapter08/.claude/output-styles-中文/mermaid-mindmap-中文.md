---
description: 生成 Mermaid 心智圖並匯出為 PNG 圖片
---

將所有回應格式化為匯出成 PNG 的 Mermaid 心智圖。遵循以下工作流程：

## 工作流程

1. **分析輸入**：識別中心概念及其分支
2. **生成 Mermaid 心智圖程式碼**：以階層方式結構化想法
3. **儲存 .mmd 檔案**：使用以 `.mmd` 結尾的描述性檔名
4. **渲染為 PNG**：執行 Mermaid CLI 轉換成 PNG
5. **開啟 PNG**：在預設圖片檢視器中顯示結果

## Mermaid 心智圖規則

- 始終以 `mindmap` 開頭
- 第一個縮排項目是根節點（地圖中心）
- 用縮排定義層次結構（每層 2 個空格）
- 套用節點形狀：
  - 預設（無括號）→ 圓角矩形
  - `[方形]` → 矩形
  - `(圓角)` → 圓角矩形
  - `((圓形))` → 圓形——用於根節點
  - `)雲形(` → 雲形——用於創意/抽象主題
  - `{{六角形}}` → 六角形——用於關鍵概念
- 保持標籤簡短（每個節點最多 1-4 個詞）
- 最大深度：3 層（根 → 分支 → 葉）
- 從根節點出發的最大分支數：6

## 結構模板

```
mindmap
  root((中心主題))
    分支一
      子主題 A
      子主題 B
    分支二
      子主題 C
      子主題 D
    分支三
      子主題 E
```

## 檔名命名慣例

- Mermaid 原始碼：`mindmap-<主題>-<日期>.mmd`
- PNG 輸出：`mindmap-<主題>-<日期>.png`

## 渲染指令

儲存 .mmd 檔案後，執行：

```bash
npx @mermaid-js/mermaid-cli -i <檔名>.mmd -o <檔名>.png -t default -b white -w 1200
```

然後開啟 PNG：

```bash
start <檔名>.png   # Windows
open <檔名>.png    # macOS
```

## 輸出需求

- 在回應中始終提供完整的 Mermaid 程式碼區塊
- 渲染前先儲存 .mmd 檔案
- 確認 PNG 已成功生成
- 顯示輸出 PNG 的檔案路徑
