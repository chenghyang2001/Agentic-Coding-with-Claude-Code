**無限代理迴圈指令**

深度思考這個無限生成任務。你即將展開一個複雜的迭代創作流程。

**變數：**

spec_file: $ARGUMENTS
output_dir: $ARGUMENTS
count: $ARGUMENTS

**參數解析：**
從 "$ARGUMENTS" 解析以下參數：
1. `spec_file` — 規格說明 Markdown 檔案的路徑
2. `output_dir` — 迭代結果存放的目錄
3. `count` — 迭代次數（1 到 N，或 "infinite" 無限模式）

**第一階段：規格分析**
讀取並深度理解位於 `spec_file` 的規格檔案。此檔案定義了：
- 要生成什麼類型的內容
- 格式與結構需求
- 特定參數或限制條件
- 各迭代之間預期的演化模式

仔細思考規格的意圖，以及每次迭代應如何在前一次的基礎上進化。

**第二階段：輸出目錄偵察**
徹底分析 `output_dir` 以了解當前狀態：
- 列出所有現有檔案及其命名規則
- 找出目前最大的迭代編號
- 分析現有迭代之間的內容演化
- 理解前幾次生成的發展軌跡
- 找出可補強或創新的空間

**第三階段：迭代策略**
根據規格分析與現有迭代：
- 確定起始迭代編號（現有最大編號 + 1）
- 規劃每次新迭代如何保持獨特性與演進性
- 思考如何在延續前次迭代的同時保持創新
- 若 count 為 "infinite"，準備持續生成直到 context 上限

**第四階段：平行代理協調**
部署多個子代理平行生成迭代，以最大化效率與創意多樣性：

**子代理分配策略：**
- count 1-5：同時啟動所有代理
- count 6-20：以每批 5 個代理分批啟動
- "infinite" 模式：以 3-5 個代理為一波，監控 context 後持續啟動新波次

**代理任務指派協議：**
每個子代理收到以下資訊：
1. **規格上下文**：完整的規格檔案分析
2. **目錄快照**：啟動時 output_dir 的當前狀態
3. **迭代編號指派**：具體的迭代編號（起始編號 + 代理索引）
4. **唯一性指令**：明確指示避免複製現有迭代的概念
5. **品質標準**：來自規格的詳細需求

**代理任務規格：**
```
任務：在 [OUTPUT_DIR] 中為 [SPEC_FILE] 生成第 [NUMBER] 次迭代

你是子代理 [X]，負責生成第 [NUMBER] 次迭代。

背景資訊：
- 規格：[完整規格分析]
- 現有迭代：[目前 output_dir 內容摘要]
- 你的迭代編號：[NUMBER]
- 指派的創意方向：[要探索的特定創新維度]

需求：
1. 完整閱讀並理解規格
2. 分析現有迭代，確保你的輸出獨一無二
3. 嚴格按照規格格式生成內容
4. 聚焦於 [指派創新維度]，同時符合規格要求
5. 以規格指定的確切命名規則建立檔案
6. 確保你的迭代帶來真正的價值與新意

交付物：單一規格指定的檔案，包含獨特且具創新性的內容
```

**平行執行管理：**
- 使用 Task 工具同時啟動所有指派的子代理
- 監控代理進度與完成狀態
- 代理失敗時重新指派迭代編號
- 確保不產生重複的迭代編號
- 收集並驗證所有完成的迭代

**第五階段：無限模式協作**
無限生成模式下，持續協調平行波次：

**波次式生成：**
1. **波次規劃**：根據 context 容量決定下一波次大小（3-5 個代理）
2. **代理準備**：為每個新波次準備全新的 context 快照
3. **漸進式精進**：每個波次應探索更進階的創新維度
4. **Context 監控**：追蹤所有代理與主協調者的總 context 使用量
5. **優雅收尾**：接近 context 上限時，完成當前波次並做摘要

**無限執行循環：**
```
當 context_capacity > 閾值時：
    1. 評估 output_dir 當前狀態
    2. 規劃下一波代理（根據剩餘 context 決定數量）
    3. 指派漸進式精進的創意方向
    4. 啟動平行子代理波次
    5. 監控波次完成情況
    6. 更新目錄狀態快照
    7. 評估剩餘 context 容量
    8. 若容量充足：繼續下一波次
    9. 若接近上限：完成最後一波並做摘要
```

**漸進式精進策略：**
- **第 1 波**：基礎功能替換，探索單一創新維度
- **第 2 波**：多維度創新，強化互動效果
- **第 3 波**：複雜範式組合，加入自適應行為
- **第 N 波**：突破性概念，挑戰規格的邊界

**Context 優化：**
- 每個波次使用全新代理實例，避免 context 累積
- 主協調者保持輕量的狀態追蹤
- 對已完成迭代進行漸進式摘要以管理 context
- 在後期波次中策略性地精簡較不關鍵的細節

**執行原則：**

**品質與獨特性：**
- 每次迭代必須真正獨特且有價值
- 在前次工作的基礎上引入新元素
- 保持與原始規格的一致性
- 確保適當的檔案組織與命名

**平行協調：**
- 策略性部署子代理，最大化創意多樣性
- 為每個代理指派不同的創新維度，避免重疊
- 協調時機，防止檔案命名衝突
- 監控所有代理確保成功完成且品質達標

**擴展性與效率：**
- 深度思考跨平行流的演化軌跡
- 無限模式下，在 context 耗盡前最大化有價值的輸出
- 使用波次式生成聰明管理 context 上限
- 平衡平行速度與品質及協調成本

**代理管理：**
- 為每個子代理提供完整的 context 與清晰的任務指派
- 優雅地處理代理失敗，重新指派迭代編號
- 確保所有平行輸出與整體進程協調一致

**深度思考指令：**
開始生成前，針對以下面向進行延伸思考：

**規格與演化：**
- 規格的深層含意
- 如何在迭代之間創造有意義的進展
- 是什麼讓每次迭代有價值且獨特
- 如何在一致性與創新之間取得平衡

**平行策略：**
- 針對請求數量的最佳子代理分配
- 如何指派不同創意方向以最大化多樣性
- 無限模式的波次大小與時機
- 多個平行代理間的 context 管理

**協調挑戰：**
- 如何防止平行流之間的概念重複
- 確保每個代理產出真正獨特輸出的策略
- 管理並發寫入的檔案命名與目錄組織
- 平行輸出的品質控管機制

**無限模式優化：**
- 持續輸出的波次生成模式
- 跨多波次的漸進式精進策略
- Context 容量監控與優雅收尾規劃
- 在平行生成速度與創新深度之間取得平衡

**風險緩解：**
- 處理代理失敗與迭代重新指派
- 確保儘管平行執行，整體進程仍保持連貫
- 管理整個系統的 context 視窗上限
- 在所有平行輸出中維持規格合規性

深度分析這些平行協調挑戰後，開始執行，系統性地完成每個階段，運用子代理實現最大化的創意輸出與效率。
